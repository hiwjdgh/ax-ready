# AI Agent Coding 준비도 산정 기준

사용자 프로젝트의 전체, 페이지, API 단위 준비도를 산정할 때 사용한다.

## 명칭

점수 이름은 "AI Agent Coding 준비도"를 사용한다.

예:

```text
전체 AI Agent Coding 준비도: 68/100
/admin/reports AI Agent Coding 준비도: 30/100
GET /api/orders/:id AI Agent Coding 준비도: 55/100
```

## 전체 프로젝트 준비도

전체 프로젝트는 100점 기준으로 산정한다.

| 항목 | 배점 |
| --- | ---: |
| 프로젝트 목적과 구조 이해 가능성 | 15 |
| 실행/빌드/테스트 명령 명확성 | 15 |
| 핵심 파일 역할 문서화 | 15 |
| Agent 작업 규칙 존재 | 15 |
| 테스트와 CI 신뢰도 | 15 |
| DB/API/domain 지식 coverage | 15 |
| 위험 영역 식별과 다음 action | 10 |

## 페이지/API 단위 준비도

페이지/API 단위는 100점 기준으로 산정한다.

| 항목 | 배점 |
| --- | ---: |
| entrypoint와 관련 파일 식별 | 20 |
| 데이터 흐름과 의존성 파악 | 20 |
| 테스트 또는 검증 방법 확인 | 15 |
| DB/API/domain 지식 확인 | 25 |
| 위험 영역과 수정 주의점 식별 | 10 |
| 부족한 정보 요청 항목 명확성 | 10 |

## 등급

- `A`: 85-100. Agent가 작은 기능/버그 수정을 비교적 안전하게 수행 가능.
- `B`: 70-84. Agent 작업 가능하지만 사람 검토와 targeted validation 필요.
- `C`: 50-69. Agent 보조 작업은 가능하나 주요 변경은 추가 지식 필요.
- `D`: 30-49. 문서화와 DB/API 지식 보강이 먼저 필요.
- `E`: 0-29. Agent coding 전환 전 구조 파악과 핵심 지식 확보가 필요.

## 점수 규칙

- 확인되지 않은 정보는 점수에 반영하지 않는다.
- DB 지식이 핵심인 기능에서 DDL 또는 stored procedure definition이 없으면 해당 페이지/API의 DB 지식 점수를 낮게 준다.
- 프론트엔드 프로젝트라도 API response schema나 상태값 의미가 불명확하면 점수를 낮춘다.
- 점수는 근거와 함께 기록한다.
- 사용자에게 추가 정보를 받은 뒤 재평가할 수 있음을 명시한다.
