# Agent Workflows

## 작은 버그 수정

1. 실패 동작을 재현하거나 위치를 찾는다.
2. 관련된 가장 작은 코드 경로를 읽는다.
3. 범위를 제한해 수정한다.
4. 대상 validation을 실행한다.
5. 변경 내용과 남는 risk를 요약한다.

## 기능 변경

1. 기존 패턴과 확장 지점을 확인한다.
2. 구현과 테스트를 함께 수정한다.
3. 동작이 바뀌면 문서나 예제를 갱신한다.
4. 영향을 받은 표면을 커버하는 validation을 실행한다.

## 리팩터링

1. 사용자가 리팩터링을 요청했는지 확인한다.
2. 동작은 바꾸지 않는다.
3. 검토 가능한 기계적 단계를 선호한다.
4. 가능하면 전후 테스트를 실행한다.

## 의존성 업데이트

1. 업데이트가 필요한 이유를 확인한다.
2. 가능하면 changelog 또는 migration note를 검토한다.
3. lockfile은 의도적으로 갱신한다.
4. build와 test를 실행한다.
