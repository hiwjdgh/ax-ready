# 기여

## 개발 흐름

1. 범위가 분명한 branch를 만든다.
2. 가장 작은 일관된 변경을 만든다.
3. 동작이 바뀌면 test를 추가하거나 갱신한다.
4. 문서화된 validation 명령을 실행한다.
5. 명확한 요약과 test note가 있는 pull request를 연다.

## Pull Request

다음을 포함한다.

- 무엇이 바뀌었는지.
- 왜 바뀌었는지.
- 어떻게 테스트했는지.
- 알려진 risk 또는 후속 작업.

## AI Agent 기여

AI-assisted change는 human maintainer가 검토하고 명확한 validation note가 포함될 때 허용된다.
